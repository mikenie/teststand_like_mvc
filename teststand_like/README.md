# 测试序列运行器

这是一个图形化的测试序列运行器，允许用户通过拖拽方式构建和执行自动化测试流程。

## 功能特性

- 自动扫描并加载以`test_`开头的Python文件中的测试函数
- 支持通过拖拽构建测试序列（包括测试函数和控制语句）
- 支持重新排序测试步骤
- 执行测试序列并输出执行日志和结果
- 提供清空序列、重载函数等操作按钮
- **新增**: 支持测试序列的保存和加载（JSON格式）
- **新增**: 配置文件管理功能（窗口位置、大小记忆等）

## 使用方法

1. 运行程序:
   ```
   python main.py
   ```

2. 点击"加载测试函数"按钮加载测试用例

3. 从左侧函数树中拖拽函数或控制语句到右侧序列列表

4. 选择序列中的步骤，在参数编辑器中设置参数

5. 点击"运行测试序列"执行整个序列，或使用"单步运行"逐步执行

## 新增功能说明

### 测试序列保存/加载

- 通过菜单栏 File -> Save Sequence 保存当前测试序列到JSON文件
- 通过菜单栏 File -> Load Sequence 从JSON文件加载测试序列
- 序列文件包含所有步骤信息、参数和输出状态

### 配置管理

- 程序会自动保存和恢复以下配置：
  - 窗口位置和大小
  - 分割面板的尺寸
  - 最后加载的序列文件路径
  - 测试目录路径

配置信息保存在程序目录下的 `config.json` 文件中。

## 目录结构

- `main.py`: 主程序入口
- `test_login.py`: 登录相关测试函数
- `test_user_management.py`: 用户管理相关测试函数
- `test_database.py`: 数据库相关测试函数
- `test_api.py`: API相关测试函数
- `README.md`: 说明文档

## 技术栈

- Python 3.x
- PyQt6

## 测试用例开发

测试用例需要满足以下规范：

1. 文件名以 `test_` 开头，以 `.py` 结尾
2. 函数名以 `test_` 开头
3. 函数可以有任意参数和返回值
4. 支持在函数中使用 `print` 输出日志
