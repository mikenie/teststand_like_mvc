# 测试序列运行器

这是一个基于PyQt6的图形化测试工具，允许用户通过拖拽方式构建和执行测试序列。

## 功能特性

1. 自动扫描当前目录下的Python测试文件
2. 在左侧面板显示可用的测试函数
3. 通过拖拽方式将测试函数添加到测试序列中
4. 提供常用的控制语句（if、for、end）用于构建复杂逻辑
5. 可以重新排列测试序列中的项目
6. 执行测试序列并查看结果

## 使用方法

### 启动应用

在命令行中运行以下命令启动应用：

```
python main.py
```

### 构建测试序列

1. 在左侧面板的函数树中，找到想要添加到测试序列的函数
2. 将函数拖拽到右侧面板的序列区域
3. 点击左侧的"If"、"For"或"End"按钮并拖拽，可以添加控制语句到序列中
4. 在序列区域内，可以通过拖拽重新排列各项的顺序

### 执行测试序列

1. 确认测试序列已按所需顺序排列
2. 点击"运行测试序列"按钮执行测试
3. 在下方的输出区域查看执行结果

### 其他操作

- 点击"加载测试函数"可以重新扫描目录中的测试函数
- 点击"清空序列"可以清除当前构建的测试序列

## 添加新的测试函数

只需在当前目录下创建新的Python文件，文件名需以"test_"开头，例如`test_example.py`，并在其中包含符合规范的测试函数：

```python
def test_example():
    """示例测试函数"""
    print("执行测试...")
    # 测试逻辑
    return True  # 返回测试结果（可选）
```

函数应满足以下要求：
1. 函数名以"test_"开头
2. 不使用下划线前缀（__function_name）
3. 函数可以有返回值（True表示成功，False表示失败），也可以没有返回值

文件命名约定：
- 所有测试文件必须以"test_"开头，以".py"结尾
- 例如：test_login.py、test_user_management.py等

## 文件说明

- `main.py`: 主应用程序文件
- `test_login.py`: 登录相关测试函数
- `test_user_management.py`: 用户管理相关测试函数
- `test_database.py`: 数据库相关测试函数
- `test_api.py`: API相关测试函数
- `README.md`: 说明文档
